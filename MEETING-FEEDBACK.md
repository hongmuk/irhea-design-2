# iRHEA-Light 디자인 리뷰 회의 피드백 정리

**회의 길이:** 약 1시간 56분
**참석자:** 7~10명 (1=PM/김 수석, 2=실무 책임자, 3=현 디자이너(HM), 5=대표님, 7=의원/유 이사, 그 외)
**다음 deliverable:** 2026-05-27 (화) — 5/25 대체 휴일(석가탄신일) 다음
**핵심 메시지 한 줄:** "1차는 매뉴얼·LCD 화면 내용을 충실히 반영. 디자인은 그 다음. 모르면 무조건 김 수석님께 물어보고, 디자이너는 장비 직접 보러 와라."

## 1. 대전제 · 진행 방향 (대표님 + 책임자 강조)

### 1.1 핵심 컨셉
- **태블릿 = 설정 + 상태 보기 도구**. 실제 추출 액션은 장비 앞 물리 버튼.
  > "왜냐하면 이게 사람이 물리적으로 준비를 해야 되거든요. 여기서 추출 커맨드를 날리지를 않도록"
  > "기본 컨셉이 얘는 설정하고 기계에 레시피를 넣어주는 것까지다. 실제 동작은 기계 앞에 가서 사람이"
- **타깃 화면 = 10인치 태블릿** (회의는 70인치 모니터로 봄)
  > "이거 어디 10인치로 봐야 되거든요. 이거 안 보여요. 노안이라서 그런지"
- **시인성 · 픽셀 사이즈 · 터치 영역**이 핵심
  > "한국 사람이 눌렀을 때 10인치 기준으로 눌렀을 때 문제는 없어야 되잖아요"
  > "돋보기 쓰고 뭐야 된다라고 하면 말이 안 되니까"
- **2천만원짜리 장비 + 20만원 태블릿 = 사용자가 보자마자 인상 쓰면 안 됨**

### 1.2 디자인 단계 분리
- **1차 (5/27 까지):** HM(EX) 매뉴얼 / LCD 화면 내용을 그대로 충실히 반영
  > "1차적으로 지금 일단 저희 거하고 거의 동일하게 한번 만들어서 1안을 주시고"
  > "매뉴얼에 있는 LCD 화면의 내용을 보면 사실 오늘 얘기한 내용들이 다 들어 있어요. 그래서 거기에 맞추는 게 일단 1차예요."
- **2차:** 안드로이드 태블릿 이점(애니메이션·스크롤·팝업) 활용한 개선
  > "기존에 LCD가 갖고 있는 한계 때문에 못했던 것들을 안드로이드라는 이점을 살리고자"

### 1.3 디자이너에게 요청
- **장비 직접 보러 오기** (수요일 또는 목요일)
  > "백날 그 앞에서 수석님이 아무리 설명을 해도요. 디자이너가 한 번 보면 이런 느낌으로 했었는데 이걸 어떻게 발전시킬까가 나오는 거지"
- **모르면 무조건 물어보기** (김 수석님)
  > "맨날 밤새운다는 얘기는 듣고 있는데 우리가 가야 할 길하고 좀 딴 길로 가는 부분이 많이 있다"
- **단순 채우기 X — 제안적이어야 함**
  > "단순하게 있는 거 그냥 딱 넣어놓고 있는 내용 다 들어갔으니까 그런 것도 아니고"
  > "트렌드가 막 이렇게 되고 있는데 옛 것을 고수하는 것도 좋지만 지금 상품인 거잖아요"
- **장비 / 데이터 / 용어 이해 부족 → 상상의 나래만 펼침** (대표님 평가)
  > "디자이너가 열심히 했는데 상상의 나래를 너무 많이 펼쳤어. 우리와 무관한 일들을 자꾸 많이 하신 것 같고"
  > "데코레이션 하려고 지금 이미지를 넣으셨잖아요. 제가 볼 때 의미 없어요."
- **디자인 시안 3개 = 사실상 1개** (바탕만 다름)
  > "3개가 다 똑같은 거였잖아요. 사실 바탕만 다르고. 디자인을 받았다고 생각을 안 하고 있어요."

### 1.4 디테일 = 품질
- **오타 · 줄 맞춤 · 폰트 강조 · 빨간색 형식 일관성** 매우 중요
  > "다 잘 하셔 놓고 이런 거 보이잖아요. 그러면 평가가 절하돼요."
- 정보 끝까지 디테일하게 정리, 한 칸 띄울 거면 일관되게

## 2. 메인 화면 (`/main`) — 가장 많이 논의됨

### 2.1 가운데 추출구 카드 (5개)
- **실사 비디오 (74 이미지) 제거** — 정신없음, 균형감 ↓, 의미 없음
  > "주변에 막 실사가 돌고 있으니까 정신이 하나도"
  > "전체적인 느낌상에서 실사가 들어가 있는 게 균형감이 좀 떨어지"
- **상단 = 전체 진행률** (퍼센트 또는 링) **/ 하단 = 단계별 표시**
  > "첫 번째 줄 또 휴지 두 번째 줄이 이렇게 넘어가잖아요. 그 단계에 맞춰서 진행이 돼야"
  > "위에 지금 표시해 놓으신 거는 이제 전체 진행률로 보는 거고요"
- **퍼센트보다 "X단계 끝" 식이 더 정확** (저울 없어서 시간 기반만 가능)
  > 대표님: "차라리 그러면 퍼센트가 아니라 그냥 2단계 끝났어 2단계 끝났어 이렇게 표시하는 게 더 낫지 않아요?"
- **단계 표시 디자인 방식**: 원 분할 / 사다리식 / 막대 — 디자이너 결정
  > "X 블룸이 약간 그런 느낌이에요. 그래프 식으로 넘어가기 좀 더 예쁘게"
  > "5개 동시에 막 이렇게 있으면은 난잡해 보일 수 있으니까 어떻게 컴팩트하게 시인성 좋게"
- **추출구 번호 (큰 숫자) 시인성** — 10인치에서 팍 띄게
- **추출 카드 상태 라벨 의미 명확화**
  > "메인 추출 추출 중인 거라기보다는 기계에 들어가 있는 거 추출 중인 거고 대기 중인 거잖아요"
- **추출 완료 시 점멸 알림** — 서빙 타이밍 (커피 식음 방지)
  > "100% 완료되면 사용자가 추출이 끝났구나라고 인지를 해야 되거든요. 그래야 서빙을 하잖아요"
- **별 표시 톤 다운** — 너무 튀지 않게
- **가운데 빈 공간 활용** (5 카테고리만큼 적정 배치)

### 2.2 추출 중인 레시피 정보 표시 (현재 카드 상단/중간)
- **4 카테고리 작은 글씨 추가**: 원두명 · 원두량 · 추출 비율 · 추출량
  > "원두명 원두량 추출 비율 추출량 그거 들어갈 수 있게 내용을 추가해 주시고요. 지금은 원두량이랑 비율만"
- **커피명 최대 2줄** (최대 60자), 긴 거는 marquee 스크롤
- **즐겨찾기 외 매뉴얼 세팅도 가능해야**
  > "즐겨찾기에서만 세팅되게 돼 있잖아요. 매뉴얼로 다른 레시피를 여기다가 선택할 수도 있어야"
  > "최종 결정된 레시피인데 꼭 즐겨찾기 있는 것만 보여지는 게 아니고 매뉴얼 세팅 가능 인터페이스까지가 저 라인에"
- **레시피 변경 시 confirm 팝업** — 잘못 누른 경우 방지
  > "기계는 지금 이게 들어가 있는데 너 진짜 이거 바꿀래?"
  > "그 멘트를 거꾸로 대표님이 저희한테 어떤 멘트로 넣는 게 좋겠습니까 라고 거꾸로 와야"

### 2.3 즐겨찾기 슬롯 (카드 하단)
- **2개 → 5~6개로 확대** (회의 중 가장 강조)
  > "즐겨찾기가 많이 나와야 돼요. 두 개만 나온 거고요. 5개 정도 한 다음에 더 많아지면 스크롤이"
  > "유저가 많이 쓰는 기능이기 때문에"
- **위로 올리고 패널식 크게**
  > "위로 올리고 6개 뭐 이렇게 5개 정도. 더 많아지면 스크롤"
  > "즐겨찾기는 크게 저 안에서 많이 나와야 돼요"
- **즐겨찾기 항목 = 레시피명만** (추가 정보 불필요)
  > "즐겨찾기에는 레시피 이름만"
  > "위에 이제 기계에 들어간 거에는 넣었는데 (4 카테고리) 즐겨찾기에는 굳이 이 내용이 없어도"
- **사용자가 즐겨찾기만 눌러 추출하는 경우 많음** — 핵심 기능
- **추출구 개수에 동적으로**: 3구 = 3줄, 5구 = 5줄
  > "3구가 접속이 되면은 이게 굳이 다섯 줄일 이유가 없잖아요. 세 줄만 표시"

### 2.4 태블릿 ↔ 장비 sync
- **유저 대기 중 / 추출 중 등 상태 노트 필요**
  > "유저가 추출 버튼을 눌러야 추출이 될 거잖아요. 추출을 안 누르고 있는 상태면은 유저 대기 중 뭐 이런 게 떠야 될 수도 있고"
- **저장 시 장비 confirm 필요** (통신 끊김/응답 없음 에러 팝업)

## 5. 사이드바 · 탑바 · 네비게이션

### 5.1 홈 버튼 위치 (참석자 1 + 2 강조, 의견 충돌)
- **참석자 1 의견**: 사이드바 메뉴는 다 하단에 있는데 홈만 위(탑바)에 있어서 위화감
  > "다 밑에 있는데 그것만 이제 근데 보통 보면은 홈은 위에 있으니까 익스플로러도 그렇고. 근데 저희는 밑에서 선택을 하게 해놨는데 홈만 위에 있잖아요. 그래서 약간 그거에 대한 위화감이 살짝"
- **참석자 2 의견**: 설정 옆에 홈 타일 추가
- **합의**: 디자이너에게 트렌드 의견 물어보고, 납득되면 위(탑바)도 OK
  > "디자이너랑 한번 얘기해 보세요. 현재 트렌드가 그래도 첫 화면에 이렇게 있는 게 낫다라고 하면은 납득이 되면은"

### 5.2 사이드바 메뉴 순서/구성
- **사이드바**: 홈 / 메인 / 레시피 / 즐겨찾기 / 정보 / **추출 실적** / 설정
  > "밑에 다 메인 그냥 한 여섯 일곱 페이지가 저희 메인 페이지들이잖아요"
- "Brewing" 메뉴는 자동 진입 (사이드바에 없어도 됨)

### 5.3 탑바 정리
- **상시 노출 = 보일러 / WiFi / 전원만**
  > "보일러랑 연결된 와이파이 연결이랑 전원 정보 정도고"
- **언어 변경 / 시간 → 하단으로** (자주 안 쓰므로)
  > "언어 변경을 해 놓으셨는데 이건 수시로 바꾸는 게 아니니까 밑으로 내려가 있어도 되고. 시간도 사실 커피 머신에서 현재 시간은 크게 중요하지 않잖아요"

## 6. 정보 / 추출 실적 페이지 분리 (대표님 지시)

### 6.1 페이지 분리 결정
- **정보 (Information) + 추출 실적 (Historical) 분리** — 페이지 1개 추가
  > "정보를 아마 저희가 두 개로 나누는 거로 했을 거예요. 그걸 정보랑 추출 실적"
  > "이력하고 실적이 페이지를 분리하는 걸로"

### 6.2 `/info` 정리
- **포함 항목**: 일련번호 · 모델번호 · 펌웨어 버전 · 최초 설치일 · 타임존 · 펌프 사용량 (추출구별) · 솔밸브 사용량 (추출구별)
- **제외 항목**:
  - **물 붓기 범위** (설정 페이지에 있음)
    > "물 붓이가 왜 들어갔는지 모르겠는데 이거는 설정 쪽에"
  - **보일러 최대 온도** (위 탑바에 표시)
  - **워터펌프 사양 · 솔밸브 종류** (불필요)
    > "워터펌프 사양을 여기다 쓸 이유도 없고요. 솔밸브도 필요 없고"
- **솔밸브 정보**: 카탈로그 X, 추출구별 사용 시간/횟수 O
  > "솔밸브에 차라리 1 2 3번 해가지고 몇 타임 펌프 몇 시간 그거가 더 중요한"

### 6.3 `/history` (추출 실적) — NEW
- **명칭**: "추출 실적" (사용자 추출 실적)
- **용도**: 사용자가 어떤 커피를 얼마나 썼는지 한눈에
  > "기간별 일별 해가지고 내가 어떤 커피를 많이 썼는지 지금까지 얼마큼을 추출했는지를 한눈에"
- **표시 항목**:
  - 토탈 추출 횟수
  - 기간별 (예: 1분기에 몇 번 레시피에 대해서 얼마큼)
  - 레시피별 통계
  - **궁극적으로 그래프** (요일별, 시간대별 인사이트)
    > "레시피 1을 딱 눌렀더니 1별로 그래프도 나올 수도 있고. 화요일 날 유독 다 많이 나갔네"
    > "사용자한테 어필하기에 되게 좋거든요"
- **정렬(Sorting) 필요** — 이름순 / 최근순 / 많은순

### 6.4 사용자 실적 초기화 = 히든 메뉴
- **펌프/솔밸브 사용 실적 초기화** = 엔지니어 모드 전용 (출고 시만 사용)
  > "사용자가 됐던 총판이 됐던 그 절대 하면 못하는 거죠. 우리만 할 수 있어요"
  > "대놓고 초기화 이게 있으면" (있으면 안 됨)

### 6.5 보안/로그/오류 = 엔지니어 모드 (숨김)
- 로그 정보 / 오류 발생 정보 = 사용자 안 봄
- 에러 발생 시 → **에러 코드만 팝업** (사용자가 보는 건 그것만)
- 엔지니어 모드 = 패스워드 진입
- **오류 발생 시 주변 동작 추출구 함께 표시** (오류 상세에)
- **로그 백업 위치 미결정** (태블릿 vs 본체 USB)

## 8. 레시피 페이지 (`/recipes`)

### 8.1 리스트 / 디스플레이
- **"12개 등록 / S6.1" → S6.1 의미 불명, 제거**
  > "s 6.1은 뭔지 모르겠어요. 불명확한 정보는 안는 이만 못합니다"
- **레시피 리스트 별 표시**: 즐겨찾기로 추출구 1~5 중 어디 들어 있는지 번호 표시
  > "레시피 사가 어디에 들어가 있는지 모르잖아요... 추출구별로 1 2 3 4 5 이렇게. 곱하기 하나가 아니라"
- **검색창 위치**: 스테이지 위 → **레시피 위로**
- **리스트 정렬(소팅)** 필요 — 이름순 / 최근순 / 많은순

### 8.2 레시피 상세 정보
- **총 추출량 (타겟) 표시 추가** — 현재 빠짐
  > "총 추출량이 없어요. 타겟 추출량 예를 들어서 20g에 16명에 320g을 추출을 하겠다는 거예요"
- **"온수 93도" 제거** — 추출 단위 카테고리에 안 맞고 보일러 95도와 불일치
  > "추출 단위에 들어갈 이유가 그 카테고리가 다른 카테고리. 보일러 위에 95도지... 내용도 안 맞고 불필요"

### 8.3 추가 / 편집 모달
- **메인 화면 항목과 일관성** (원두명 누락 등)
  > "메인 화면의 레시피 항목과 추가/편집 모달의 항목이 다름"
- **필수 항목**: 레시피명, 원두명 — 안 채우면 저장 X (validation)
  > "그렇게 했을 경우에 저장을 누르면은 그게 안 채워졌으면은 채워야지 저장이 된다라는 멘트가 떠야"
- **🚨 가장 중요한 빠진 부분: 스테이지 편집 기능**
  > "팝업이 예를 들어서 레시피 추가인데 이건 사실 별 중요하지 않은 정보예요. 얘는 기본 정보인 거고 실제 레시피는 옆에 있는 스테이지예요"
  > "원두량이랑 비율만 중요해요. 그 정보를 가지고 어떤 추출 레시피를 짤지가 그 스테이지예요. 근데 지금 정작 중요한 스테이지 편집 부분이 빠져 있어요"
- **스테이지**:
  - 최대 10단계
  - **유속 = 정수만** (1, 2, 3 — 소수 X)
  - **마지막 스테이지 자동 계산** 또는 합계 표시
    > "30 30 20을 했어요. 그럼 80이잖아요. 마지막에 이제 직접 20을 넣을 수도 있지만은 자동 계산"
  - **스테이지 추가 시 자동 합계** — 그람 단위 헷갈림 방지
    > "스테이지를 추가하게 추가할 때마다 한 칸씩 밑으로 밀리면 되죠. 칸이면 다섯 번째 칸이 그 총합이라는"
  - **물 붓기 168 = pour rate** (소수점 명시 필요)
- **모달 vs 인라인 결정 필요** — 디자이너 검토
- **레시피명/디스크립션 분리 제목 불필요** (한 화면에 나오면)
- **테스트 추출 버튼**: 메인 화면에서 빠지고 → 편집/추가 모달에만
  > "이 페이지 메인이 되는 레시피 페이지에서 테스트 추출은 빠져야 돼요"
- **테스트 추출 동작**: 빈 추출구 선택 → 임시 추출 → 저장 여부 판단
  > "1번 3번이 비어 있어요. 그러면 얘를 1번에다 넣고 테스트 추출을 해볼 거냐 그 선택도 필요"

### 8.4 삭제 / 다중 선택
- **OX 모달 → 직관적 버튼** (확인/취소 등)
  > "복구 불가 진행하시겠습니까? 이렇게 ox로 나오는데 오스 조금 아닌 것 같아요"
- **다중 선택 시**: 삭제만 가능 / 편집·추가 막기
  > "다중 선택했는데 편집 누르면 어떡해. 다중 선택은 삭제만"
- **클라우드 다운로드** = 다중 선택 가능

### 8.5 레시피 풀 컨셉 변경 (기존 vs Light)
- **기존**: 추출구별 레시피 (다를 수 있었음)
- **Light**: 전체 레시피 풀 (모든 추출구 동일)
- **스테이지별 온도 세팅 불가능** (보일러 1개)

### 8.6 클라우드
- **다운로드 + 업로드 양방향**
- **MCP 컴이 클라우드 연동** (태블릿 아님)
- **WiFi 2개 사용 결정** (확장 USB 슬롯) — 미결정

  > "장비 설정으로 해도 될 것 같기도 하고 하이어라키컬하게 그 이름도 정리"

### 10.2 제거할 항목

### 10.3 보일러 정보
- **"웜업 상태" 제거** — 장비 켜진 시간과 동일
  > "목표 온도에 도달하면 계속 켜져 있는 거니까 장비 켜져 있는 시간이랑 보일러 켜져 있는 시간 동일"
- **장비 동작 전체 타임** 그렇게 중요 X

### 10.4 워터린스
- **가독성 개선** — 제목 볼딕, 값 옅게
- **현재 가로/세로 매칭 헷갈림** — 정리 필요
  > "온수량이 빠진 것 같네 하고 자세히 봤더니 최대 값이 있어요. 이건 정리가 필요"
- **워터린스 온도 = 보일러 설정 온도 가져오기** (보일러 95도면 85도 못 내림)
  > "보일러가 하나인데 위에는 95도인데 밑에 85도 물을 내릴 수가 없어요. 불필요한 정보"

### 10.5 모든 설정 값
- **min/max validation 팝업** ("최대 OO입니다")
  > "최대가 55인데 60이 들어가면은 그냥 55가 들어가게 되어 있어요. 최댓값은 이거다라는 멘트가 또 떠야"
- **물 붓기 범위**: 25~55
- **유속**: 3~5 (현재 6mm는 범위 초과)
- 모든 항목 가이드 표시 + 한계 팝업

### 10.6 🚨 빠진 항목: 레시피 기본 디폴트 값 설정
- **추가 필요** — 레시피 추가 시 자동으로 채워질 디폴트 값
  > "레시피를 추가를 딱 하잖아요. 그럼 디폴트 값이 자작자작해서 나올 거예요. 다 0이 아니고 그 디폴트 설정을 여기서 하게 만들어 놔야"
- **이유**: 카페 한다고 커피 잘 아는 거 아님 → 가이드 필요
- **저장 위치**: 클라우드에 저장 → 원격 모니터링용
  > "기본 설정 값을 보고 장비 설정이 이상해요라고 사용자분이 잘못 설정하셨네"

### 10.7 저장 시 confirm 팝업
- **변경 사항 있을 때 무조건 confirm** (저장 / 취소)
  > "저장 취소는 무조건 팝업 띄우세요. 뭔가 하나 바뀐 게 있다 하면은 다 띄우기를 원합니다"
- **이유**: 실수 방지 ("후회할 때 많잖아요")

## 12. 펌웨어 업그레이드 (`/settings/firmware`)

### 12.1 엔지니어링 vs 펌웨어 관계
- **펌웨어 = 사용자용** (비밀번호 없음)
- **엔지니어링 = 관리자용** (비밀번호 있음)
- **탭 순서 변경 필요**: 일관성 깨짐
  > "쭉쭉 넘어가다가 막히고 다시 풀리고 이상하잖아요"
- **엔지니어링 탭 → 맨 뒤 또는 hidden**
  > "엔지니어링 메뉴를 히든으로 뽑아도 될 것 같기도"
- **탭 순서**: 설정 → 백업 → 펌웨어 → (엔지니어링 또는 hidden)

### 12.2 업그레이드 진행 표시
- **이미지적으로** (단순 숫자 X)
  > "이미지적으로 이렇게 가 있으면 좋아 그냥 단순 숫자로 넘어가는 것보다는"
- **보드별 진행 표시** — 설치 프로그램 스타일
- **보드 종류**: WHP / MCP / 5천원 / 백 프론트
  > "WHP 하나 있고 MCP 컴 5천원 빼고 백 프론트 있으면 그것들도 같이 나와야"
- **퍼센트 + 어떤 파일 진행 중**

### 12.3 업데이트 확인 정책
- **AWS 자동 + USB 수동** 둘 다 필요
- **순서**: 설치 방식(자동/수동) 선택 → 자동일 때만 정책 노출
  > "수동으로 하게 되면은 다 필요 없는 거고 자동으로 하게 되면은 하루에 한 번 할 건지 전원 켤 때 할 건지"
- **정책 옵션**: Always check / Once / Never
- **위치 바꿔야** — 설치 방식 먼저, 정책 나중
- **USB vs AWS 디렉션**: USB 누르면 USB 체크, AWS 누르면 AWS 체크

  - 페이지·팝업·패스워드 등 모든 분기 표시
  - 대표님이 빠진 거 / 추가할 거 체크 가능
  > "한눈에 볼 수 있는 한눈에 볼 수 있는 이런 걸 원합니다"
- **장비 직접 보러 오기** (수요일 또는 목요일)
- 1차 내용 반영 → 2차 디자인

### 15.4 미결정 사항 (대기)
- WiFi 2개 USB 허브 사용 여부
- 로그 백업 위치 (태블릿 vs 본체 USB)
- 추출구별 즐겨찾기 슬롯 적정 갯수 (태블릿 실측 후)
- 진행 표시 디자인 (원형 vs 사다리 vs 막대)
- AWS 업데이트 정책 옵션 확정
